# 功能核对报告

根据 `功能核对.md` 核心需求，对 Finer PSI 进销存系统进行功能核查，结果如下。

---

## 1. 使用端

| 需求 | 状态 | 说明 |
|------|------|------|
| 电脑网页端 | ✅ 有 | Vue 2 + Ant Design Vue 前端，响应式布局 |
| 手机微信端 | ✅ 有 | H5 网页，微信内可直接打开，无需下载 APP |

**结论**：PC 端与微信 H5 端均支持，满足「电脑网页端 + 手机微信端」需求。

---

## 2. 角色权限

| 需求 | 状态 | 说明 |
|------|------|------|
| 老板（全权限、审批） | ✅ 有 | 预设角色「老板」(psi-boss)，复制单据审批者权限；付款申请/付款单支持 BPM 审批 |
| 财务（制单/复核/记账） | ✅ 有 | 预设角色「财务」(psi-finance)，应收应付、收款付款、发票、核销等模块；支持制单、审核、记账 |
| 仓管（出入库/库存管理） | ✅ 有 | 预设角色「仓管」(psi-warehouse)，出入库单、盘点、库存、调拨等模块完整 |
| 账号新增、禁用 | ✅ 有 | 用户管理（UserList）支持新增、禁用 |
| 权限分配 | ✅ 有 | 角色管理、菜单权限、数据权限 |
| 操作日志可追溯 | ✅ 有 | 系统日志（LogList）、数据日志（DataLog） |

---

## 3. 核心模块

### 3.1 对公付款中心

| 需求 | 状态 | 说明 |
|------|------|------|
| 付款申请 | ✅ 有 | `FinPaymentReq` 付款申请单，支持采购付款申请、采购预付申请、其他付款申请 |
| 收款方信息 | ✅ 有 | 通过供应商（`bas_supplier`）关联，含办款资料（单位、开户行、账号等） |
| 金额、用途 | ✅ 有 | 申请金额、已付金额；用途可写在明细/备注 |
| 附件上传 | ✅ 有 | 付款申请单表 `fin_payment_req` 有 `attachment` 列；前端各付款申请表单（采购付款申请、采购预付申请、其他付款申请等）均有 `j-upload` 附件上传组件 |
| 多级审批 | ✅ 有 | 内置 BPM，付款申请、付款单等支持发起审批、结束审批 |
| 一键跳转建行企业银行 | ✅ 有 | 付款申请/付款单列表工具栏新增「跳转建行企业银行」按钮，新窗口打开 https://b2bbank.ccb.com |
| 付款记录归档 | ✅ 有 | 付款单（`FinPayment`）、付款申请归档可查 |
| 回单上传 | ✅ 有 | 付款单表 `fin_payment` 有 `attachment` 列；各付款单表单均有「附件」上传组件，可上传银行回单 |

### 3.2 财务管理

| 需求 | 状态 | 说明 |
|------|------|------|
| 收支记账 | ✅ 有 | 收款单（FinReceipt）、付款单（FinPayment）、其他收支 |
| 应收应付管理 | ✅ 有 | 应收单、应付单、应收核销、应付核销 |
| 发票登记 | ✅ 有 | 采购发票（FinPurInvoice）、销售发票（FinSalInvoice） |
| 收支报表统计 | ✅ 有 | 应收应付明细、月汇总、余额等统计 |
| Excel 导出 | ✅ 有 | 各列表支持 `exportXls` 导出 |

### 3.3 进销存管理

| 需求 | 状态 | 说明 |
|------|------|------|
| 水产物资分类管理 | ✅ 有 | `bas_material_category` 物料分类，`bas_material` 物料；分类可自定义（虾苗/饲料/渔药等） |
| 采购入库 | ✅ 有 | 采购入库单（PurIn）、红字采购入库 |
| 销售出库 | ✅ 有 | 销售出库单（SalOut）、销售退货入库 |
| 库存预警 | ✅ 有 | 物料支持安全库存字段；库存预警列表、首页入口 |
| 盘点记录 | ✅ 有 | 盘点卡（StkCheck）、盘盈入库、盘亏出库 |

### 3.4 往来单位管理

| 需求 | 状态 | 说明 |
|------|------|------|
| 客户信息维护 | ✅ 有 | `bas_customer` 客户表及维护界面 |
| 供应商信息维护 | ✅ 有 | `bas_supplier` 供应商表及维护界面 |
| 付款/出入库快速调取 | ✅ 有 | 单据中选择供应商、客户，字典联查 |

### 3.5 老板数据看板

| 需求 | 状态 | 说明 |
|------|------|------|
| 首页展示 | ✅ 有 | `/dashboard/analysis` 首页 |
| 今日/本月收支 | ✅ 有 | 首页新增「今日/本月收支」卡片，展示收款、付款的今日笔数/金额、本月笔数/金额 |
| 待审批付款 | ✅ 有 | 卡片右上角显示「待审批付款 X 笔」入口，表格「待核批」列可点击跳转采购付款申请列表 |
| 库存预警 | ✅ 有 | 首页展示预警数量，可点击进入库存预警列表 |
| 应收应付总额 | ✅ 有 | 首页「业务概况」展示客户欠款(应收)、欠供应商(应付) |
| 快捷操作按钮 | ⚠️ 部分 | 有菜单导航，无针对性快捷入口 |

**结论**：老板看板已有业务概况（应收应付）、销售采购统计、处理中单据（含付款申请待核批）；需补充今日/本月收支流水、待审批付款单独入口、库存预警及快捷操作。

**核查要点**：实际首页使用 `Analysis.vue`（业务看板），通过 `home_summary`、`home_sale`、`home_purchase`、`home_doing_bill` 等报表接入真实数据，而非 `IndexChart.vue` 的通用模拟数据。

## 7. 结论

| 分类 | 满足度 | 说明 |
|------|--------|------|
| 角色权限 | 高 | 基本满足 |
| 财务管理 | 高 | 收支、应收应付、发票、报表、导出均具备 |
| 进销存 | 高 | 库存预警已补齐 |
| 往来单位 | 高 | 客户、供应商维护完整 |
| 对公付款 | 高 | 付款申请、审批、建行跳转、附件/回单上传均有 |
| 老板看板 | 较低 | 首页为 Analysis 业务看板，已有应收应付、销售采购统计、处理中单据；缺收支流水、待审批入口、库存预警、快捷按钮 |

**整体**：核心进销存与财务功能较完整，对公付款和老板数据看板需补充与定制。
