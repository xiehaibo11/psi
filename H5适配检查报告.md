# H5/移动端适配检查报告

## 一、已完善项

### 1. Viewport 与 Meta
- **viewport**：`width=device-width, initial-scale=1.0, maximum-scale=5.0, user-scalable=yes`
  - 允许用户缩放，支持无障碍
  - 最大缩放 5 倍，兼顾可用性与可访问性
- **format-detection**：`telephone=no`，避免系统自动识别号码为链接
- **pragma**：修正 `pragram` → `pragma` 拼写错误

### 2. 设备检测
- `enquire.js`：≤1088px 视为 tablet/mobile，≤768px 视为手机
- `App.vue` 初始化时调用 `ToggleDevice`
- 使用 `mixinDevice` 的 `isMobile()` / `isDesktop()` 做响应式

### 3. 布局与导航
- **侧边栏**：移动端使用 Drawer 抽屉
- **顶栏**：移动端显示菜单图标、当前菜单标题、UserMenu
- **响应式栅格**：Analysis 等页面使用 `a-col` 的 xl/lg/md/sm/xs

### 4. UserMenu 移动端
- 搜索：移动端弹窗形式
- 用户下拉：含欢迎语、当前月度、切换部门、密码修改等
- **系统设置**：移动端也可通过下拉菜单进入（已开放）
- **暗黑模式切换**：顶栏图标，移动端可用

### 5. 首页 Analysis 移动端
- 业务概况：`isMobile()` 时单列、居中
- 各卡片：使用 `:sm="24" :xs="24"` 全宽
- **处理中主要单据**：移动端 `min-height: 200px`，替代原 824px，减少空白

### 6. 表格
- 移动端取消 `min-width`，支持纵向滚动（JTC-480）
- `scroll="{x: true}"` 保留横向滚动能力

### 7. 其他
- 登录页使用 `mixinDevice`，按钮全宽
- 打包 App 场景：`plusready` 监听返回键，双击退出
- `polyfill` 已引入，兼容旧设备

## 二、建议进一步优化（可选）

| 项 | 说明 |
|----|------|
| 触摸热区 | 关键按钮建议保持 ≥44px 点击区域（当前 action padding 约 12px） |
| 长列表 | 如采购/销售单据列表，可评估虚拟滚动 |
| 图表 | Bar 等图表在小屏上的可读性可再微调 |
| 安全区域 | 如需适配刘海屏，可增加 `viewport-fit=cover` 和 `env(safe-area-inset-*)` |
